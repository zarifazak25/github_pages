Part 1: Setting Up Your Live Website

From Zero to Published in 7 Steps

In this section, you'll create your GitHub account, choose a template, and publish your website live on the internet. By the end of this part, you'll have a working website with its own URL!

Step 1: Create Your GitHub Account

GitHub is a platform where developers store and share code. It's also where you'll host your website for free!

What to do:

  1. Open your web browser and go to GitHub.com
  2. You'll see a "Sign up" button in the top right corner. Click it.
  3. Choose one of these sign-up methods:
    • Sign up with Google: If you have a Gmail account, this is the fastest option. Click the "Sign up with Google" button and follow the prompts.
    • Sign up with Email: Enter your email address, create a password (make it strong!), and choose a username. Your username will be part of your website URL, so choose something professional.
  4. Complete the verification process (you might need to solve a puzzle or verify your email)
  5. Follow any additional setup steps GitHub shows you
üí° Note: If you already have a GitHub account, simply log in and skip to Step 2.
‚ú® Tip: Your GitHub username will appear in your website URL like this: https://your-username.github.io. Choose a professional username if you plan to use this website for your portfolio or resume!

Step 2: Choose a Website Template

Instead of building a website from scratch, you'll start with a professionally designed template. This gives you a beautiful starting point that you can customize later.

What to do:

  1. Open this link in a new tab: Template Gallery
  2. Browse through the available templates. Each template shows a preview of what the website looks like.
  3. Consider these factors when choosing:
    • Purpose: Are you making a portfolio, resume site, or personal blog?
    • Style: Do you prefer minimalist, colorful, modern, or professional designs?
    • Layout: Do you want a one-page site (everything on the homepage) or multi-page site (separate pages for About, Projects, Contact, etc.)?
    • Features: Does it have sections you need like portfolio gallery, blog, contact form, etc.?
  4. Click on templates to see larger previews and explore their features
  5. Take your time! You want to pick something you'll be happy with.
‚ö†Ô∏è Important: Choosing your template is an important decision! If you want to change templates later, you'll have to start over from Step 1 and rebuild your repository. So take your time and choose one you'll stick with.
‚ú® Tip: Look for templates that match your goals. If you're a designer, choose one with a strong portfolio section. If you're a writer, pick one with a blog layout. If you want to showcase projects, find one with a projects gallery.

Step 3: Access the Template Repository

Once you've chosen your template, you need to access its code repository (the place where all the website files are stored).

What to do:

  1. On your chosen template card, look for the black button that says "GitHub Repo"
  2. Click this button
  3. You'll be redirected to GitHub, where you'll see the template's repository page
  4. This page shows all the code files that make up the website. Don't worry if it looks complicated‚Äîyou'll learn what everything means soon!
üì∏ Screenshot: Template card showing the black "GitHub Repo" button
üí° What is a repository? A repository (or "repo" for short) is like a folder that holds all the files for a project. In this case, it holds all the files for your website: HTML, CSS, images, and more.

Step 4: Create Your Repository

Now you'll make your own copy of the template. This is called "creating a repository from a template."

What to do:

  1. At the top of the repository page, look for a green button that says "Use this template"
  2. Click it, and a dropdown menu will appear
  3. Select "Create a new repository"
  4. You'll now see a form with several fields to fill out:

Filling out the repository form:

A. Repository Name:

  • Enter a name for your website repository
  • Good examples: my-portfolio, john-doe-website, personal-site
  • Bad examples: website 123 (has spaces), My_RANDOM_Site!!! (unprofessional)
  • Use lowercase letters, hyphens (not spaces), and keep it simple
üì∏ Screenshot: Repository name field with example name entered

B. Description:

  • Add a brief description of yourself or your website
  • Example: "Personal portfolio showcasing my web development projects and skills"
  • Example: "Professional resume site for Jane Doe, UX Designer"
  • This description helps with SEO (Search Engine Optimization), which means Google and other search engines can find your website more easily

C. Visibility:

  • Make sure "Public" is selected
  • This is crucial! If you select "Private," your website won't be accessible to anyone on the internet
‚ö†Ô∏è Critical: Your repository MUST be set to "Public" for GitHub Pages to work. A private repository won't create a live, publicly accessible website.
  1. Once you've filled everything out, click the green "Create repository" button at the bottom
  2. Wait a few seconds while GitHub creates your repository
üéâ Success! You now have your own copy of the template! You'll be redirected to your new repository's homepage.
‚ú® SEO Tip: A well-written description with relevant keywords helps people find your website through Google. Include words related to your profession, skills, or what visitors will find on your site.

Step 5: Understanding Your Repository Structure

Before we make the website live, let's take a quick tour of what you're looking at.

The Repository Homepage:

You're now looking at your repository homepage. Here's what you'll see:

  • The <> Code tab: This is where you'll spend most of your time. It shows all your website files.
  • File List: You'll see a list of files and folders. These are all the pieces that make up your website.

Essential Files You'll See:

1. HTML Files (.html):

  • HTML stands for "HyperText Markup Language"
  • HTML files contain the structure and content of your web pages‚Äîthe text, headings, paragraphs, links, etc.
  • index.html is your homepage (the main page people see when they visit your site)
  • If you have a multi-page website, you'll see other HTML files like about.html, projects.html, etc.

2. CSS Files (.css):

  • CSS stands for "Cascading Style Sheets"
  • CSS files control the design and appearance of your website‚Äîcolors, fonts, spacing, layouts, etc.
  • Common names: style.css, main.css, styles.css
  • Without CSS, your website would be plain black text on a white background!

3. Other Files You Might See:

  • JavaScript Files (.js): These add interactive features (optional, not all templates have them)
  • Image Folder (/images or /img): Stores photos, icons, and graphics
  • Documents Folder (/files or /documents): Stores PDFs like your resume
  • README.md: A text file explaining the template (useful to read!)
üí° Quick Analogy: Think of building a website like building a house:
  • HTML is the structure (walls, rooms, doors)
  • CSS is the decoration (paint colors, furniture, style)
  • JavaScript is the functionality (light switches, appliances)
‚ú® Tip: Don't worry if you see files you don't recognize yet! You don't need to understand everything right now. In Part 2, we'll focus only on the files you need to edit.

Step 6: Make Your Website Live (Enable GitHub Pages)

This is the most important step! Right now, your repository exists, but it's not a live website yet. This step activates GitHub Pages, which turns your code into a real website that anyone can visit.

What to do:

  1. At the top of your repository page, look for the navigation bar with tabs like Code, Issues, Pull requests
  2. Click on Settings (it's usually the last tab on the right)
  3. You'll see a sidebar on the left side of the screen
  4. In the sidebar, scroll down until you find a section called "Code and automation"
  5. Under "Code and automation," click on Pages
üì∏ Screenshot: Settings page showing the left sidebar with "Pages" highlighted

Configuring GitHub Pages:

  1. On the Pages settings screen, look for a section called "Source" or "Build and deployment"
  2. Under "Branch", you'll see a dropdown that says "None"
  3. Click on it and select "main" (some templates might say "master" instead‚Äîthat's okay, choose that)
  4. After selecting "main," another dropdown will appear next to it showing a folder icon
  5. This should automatically say "/ (root)"‚Äîleave it as is
  6. Click the Save button
üì∏ Screenshot: Pages settings showing Branch dropdown with "main" selected and folder set to "/ (root)"
üí° What just happened? You told GitHub: "Please take the files in the 'main' branch of my repository and publish them as a website!" GitHub Pages will now process your files and create a live website.

Waiting for Your Site to Go Live:

  1. After clicking Save, GitHub starts building your website. This takes 2-3 minutes
  2. You'll see a message that says "GitHub Pages source saved" or something similar
  3. Be patient! Don't refresh constantly. Give it a full 2-3 minutes.
  4. After waiting, refresh the page (press F5 or click the refresh button in your browser)
üéâ Your Website is Live! After refreshing, you'll see a message at the top that says:

"Your site is live at" https://your-username.github.io/repository-name/

Viewing Your Website:

  1. You'll see a "Visit site" button or clickable link
  2. Click it to open your live website in a new tab
  3. Congratulations! üéä You now have a real website on the internet with its own URL!
‚ú® Share Your Site: You can now share this URL with anyone in the world, and they'll be able to see your website! Copy the URL and test it in a different browser or on your phone.
‚ö†Ô∏è Troubleshooting: If your site isn't showing up after 5 minutes:
  • Make sure you selected "main" branch (not "None")
  • Verify your repository is set to "Public" (check Settings ‚Üí General ‚Üí Danger Zone)
  • Try refreshing the Settings ‚Üí Pages page again
  • Clear your browser cache and try the URL in an incognito/private window

Step 7: Quick Access Setup

This final step makes your workflow easier. You'll add your website URL to your repository homepage so you can quickly visit your live site without going to Settings every time.

What to do:

  1. Copy your website URL: Go back to Settings ‚Üí Pages and copy the full URL of your live website (e.g., https://your-username.github.io/repository-name/)
  2. Go to your repository homepage: Click the <> Code tab at the top left
  3. Find the "About" section: On the right side of the page, you'll see a section labeled "About" (it's in the sidebar)
  4. Click the settings gear icon: Next to "About," there's a small ‚öôÔ∏è gear/settings icon. Click it.
  5. Add your website URL: A popup will appear. Look for a field labeled "Website" and paste your website URL there
  6. Save changes: Click "Save changes" at the bottom of the popup
üì∏ Screenshot: Repository homepage showing the About section with the settings gear icon highlighted
üì∏ Screenshot: About settings popup showing the Website field with URL pasted in
üéâ Done! Now your website URL appears in the "About" section of your repository. You can click it anytime to quickly visit your live site while you're editing!
‚ú® Pro Tip: This is incredibly useful when you're making changes to your website. You can:
  1. Edit files in the Code tab
  2. Save changes (commit)
  3. Click your website link in the About section to see the changes live
  4. No need to dig through Settings every time!
